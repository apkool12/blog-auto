# 오늘 한 일 정리 (Blog Auto)

Git 커밋과 변경 내역을 **블로그용 초안**으로 정리하는 Cursor/VS Code 확장입니다.

**저장소**: [github.com/apkool12/blog-auto](https://github.com/apkool12/blog-auto)  
티스토리·벨로그 등에 올리기 전에 "오늘 한 일"을 빠르게 모아서, Cursor로 요약·말투까지 적용할 수 있게 돕습니다.

## 기능

- **커밋 기준 정리**: 선택한 기간(오늘/어제/지난 7일/직접 입력)과 브랜치의 커밋 + 변경 파일 통계를 한 번에 정리
- **핵심 코드(diff) 포함**: 옵션으로 커밋별 실제 코드 변경 일부를 포함해 Cursor 요약 품질 향상
- **기술·진행 요약 요청**: 정리본을 Cursor에 붙여넣을 프롬프트로 복사 → 사용 기술·어려움·진행 내용 요약
- **말투 학습**: 블로그 URL로 말투 샘플 불러오기 → "Cursor로 말투 적용"으로 내 스타일로 재작성 요청
- **초안 저장/불러오기**: 정리한 내용을 저장해 두었다가 나중에 불러와서 수정·발행
- **메모 파일 병합**: 설정에 지정한 메모 파일(예: `memo/today.md`) 내용을 정리본 맨 뒤에 "오늘 메모"로 붙이기
- **설정**: 제목 형식, 인트로 문장, diff 줄 수·포함 확장자·제외 패턴 등 설정에서 변경 가능

## 요구 사항

- **Cursor** 또는 **VS Code** 1.85+
- **Node.js** 18+ (개발/빌드 시)
- Git 저장소가 있는 워크스페이스(커밋 정리 사용 시)

## 설치

### .vsix로 설치 (배포용 패키지)

1. 저장소에서 `npm run package` 실행 → `blog-auto-0.1.0.vsix` 생성
2. Cursor/VS Code에서 **확장** 뷰 → `...` 메뉴 → **VSIX에서 설치** → 생성한 `.vsix` 선택

## 사용법

1. **탐색기** 사이드바에서 **오늘 한 일 정리** 뷰 열기
2. **기간** 선택(오늘/어제/지난 7일/직접 입력) → **브랜치 불러오기** → 정리할 브랜치 선택
3. (선택) **핵심 코드 변경(diff) 포함** 체크 → **선택 기간·브랜치로 정리** 클릭
4. **기술·진행 요약 요청** → Cursor 채팅(Ctrl+L)에 붙여넣기 → 요약 받기
5. (선택) **말투 샘플 불러오기** 후 **Cursor로 말투 적용** → 초안을 내 블로그 말투로 재작성 요청
6. **초안 저장** 해 두었다가, 수정 후 **새 문서로 열기** 또는 **클립보드에 복사**로 블로그에 붙여넣기

## 설정 (블로그 정리)

| 설정                                            | 설명                                                |
| ----------------------------------------------- | --------------------------------------------------- |
| `blogUrl`                                       | 말투 학습용 블로그 주소                             |
| `postTitleFormat`                               | 제목 형식. `{{date}}` 사용 가능                     |
| `introTemplate`                                 | 인트로 문장. `{{branch}}`, `{{branches}}` 사용 가능 |
| `memoFilePath`                                  | 오늘 메모 파일 경로 (워크스페이스 기준)             |
| `diffMaxLinesPerCommit` / `diffMaxLinesPerFile` | diff 줄 수 제한                                     |
| `diffIncludeExtensions`                         | diff에 포함할 확장자 (쉼표 구분)                    |
| `diffSkipPatterns`                              | diff에서 제외할 경로 패턴 (쉼표 구분)               |

## 개발

```bash
npm install
npm run compile
# Cursor/VS Code에서 F5 → 확장 디버그 실행
```
