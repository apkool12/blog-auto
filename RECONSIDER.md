# 재검토 포인트 (나중에 다시 생각해 보기)

플러그인을 쓰다 보면 "이렇게 바꾸면 더 블로그에 맞겠다" 싶은 부분이 생길 수 있어요. 그때 수정하기 쉽게, 지금 설계에서 의도적으로 둔 부분과 나중에 손볼 만한 지점을 적어 둡니다.

- **제목/인트로 템플릿**: 지금은 설정에 `{{date}}`, `{{branch}}`, `{{branches}}`만 치환해요. "섹션 순서(기술 → 어려움 → 진행)"를 템플릿으로 바꾸거나, 여러 템플릿 프리셋을 두는 건 아직 없어요. 필요하면 `postTitleFormat` / `introTemplate` 옆에 `sectionOrder` 같은 설정을 추가하면 됩니다.
- **에러 메시지**: `MSG` 상수로 모아 두었어요. 톤을 더 부드럽게 하거나, "이렇게 해보세요" 링크(문서/명령)를 붙이면 좋을 수 있어요.
- **초안 저장 위치**: 지금은 `globalState`(전역)라 워크스페이스가 바뀌어도 같은 목록이에요. "워크스페이스별로 저장"이 필요하면 `workspaceState`나 워크스페이스 내 파일로 바꾸면 됩니다.
- **날짜 "직접 입력"**: UI는 시작/끝만 있어요. 타임존은 git `--since`/`--until`이 ISO 문자열을 쓰므로 현재대로 두었고, "로컬 날짜" 표시가 중요해지면 그때 보완하면 됩니다.

이 파일은 참고용이에요. 실제로 다시 작성할 때는 이 목록을 보면서 한 항목씩 바꿔 나가면 됩니다.
